# Отчёт о найденных дефектах в StringUtils

Баг 1: Метод trim удаляет только один вид пробела**

Описание:  
Метод trim удаляет только символ пробела ' ', но не реагирует на табуляцию (\t), перенос строки (\n) и другие пробельные символы.

Пример:  
trim("\t\nhello") → "\t\nhello" (не меняется)

Ожидаемо:  
Должен удалять все пробельные символы в начале, как str.lstrip().

Рекомендация:  
Использовать string.lstrip() или расширить список удаляемых символов.

Статус: Нужно исправить

Баг 2: Метод contains возвращает True для пустого символа**

Описание:  
contains("hello", "") → True

Причина:  
В Python пустая строка содержится в любой строке (включая пустую). Это может быть неочевидно для пользователя.

Рекомендация:  
Уточнить в документации или добавить проверку: если symbol == "", возвращать False.

Статус: Требует уточнения требований

Баг 3: Метод to_list не удаляет пробелы вокруг элементов**

Описание:  
to_list("a, b, c") → ["a", " b", " c"] — пробелы остаются.

Ожидаемо:  
["a", "b", "c"] — как в большинстве CSV-парсеров.

Рекомендация:  
Добавить опцию удаления пробелов или использовать strip() при разборе.

Статус: Нужно улучшить
